# Small Forums List

一个整理国内外小型、圈内、传统论坛和隐藏社区入口的列表。

这里的“小型”不是严格按用户量计算，而是指：如果你不主动搜索、不混相关领域，基本不会自然刷到的社区。它们可能是技术论坛、主机圈、ACG 小圈子、站长生态、硬件论坛、成人/资源向灰区社区，也可能是海外老互联网论坛。

## 说明

- 目前主要由 AI 按公开信息、站点页面和少量个人体验整理；只有一部分社区是我真实长期进入体验过的。
- 欢迎各社区成员提交 PR，修正入口、分类、注册方式，或者补充你对社区氛围和真实体验的看法。
- 收录不是推荐、背书或导流，只是做互联网社区地图。
- 成人、资源、破解、逆向、安全、刷机、交易等内容会按大类写清楚，但不写具体操作路径。
- 未长期使用过的社区，会尽量写成公开信息和圈内观察，不假装成亲身体验。

## 技术 / AI / 开发者

### Linux.do

- 入口：https://linux.do/
- 定位/氛围：根据我的实地体验，Linux.do 是节奏很快的中文技术/AI 社区：AI、部署、Cloudflare、VPS、域名、公益站、工具情报和日常茶馆混在一起，信息密度高，规则也比普通论坛严格。
- 注册：当前以站内规则和邀请/认证通道为准；新人建议先读 guidelines，完成 bot 新手任务，确认升到 1 级再发言。GitHub 5 年及以上老账号曾可走 invite.linuxdo.org 认证通道，具体以官方页面为准。
- 适合看/里面有什么：适合看 AI 公益站列表、模型/工具讨论、Cloudflare/VPS/域名部署教程、福利羊毛、开发调优、文档共建、搞七捻三日常贴，以及 credit.linux.do 等社区积分/服务玩法。
- 注意：新人期建议先只读、做新手任务、熟读站规，尤其注意站内对抽奖、广告和交易内容的要求。常见经验是先升到 1 级，长期活跃到 2 级会更稳；涉及广告、交易或金钱相关内容时按站内分组和规则处理。不要购买来路不明的邀请码，也不要把短期羊毛当长期服务。
- 来源：https://linux.do/、https://linux.do/guidelines、https://invite.linuxdo.org/、https://credit.linux.do/
- 核验：2026-06-22

## 贡献方式

新增社区时请同时更新：

1. `data/communities.json`
2. 运行 `node scripts/generate-readme.mjs` 同步 README
3. 必要时更新 `docs/categories.md`
4. 运行 `node scripts/build-site.mjs` 预览生成页面

条目描述尽量写清楚：入口、社区氛围、主要内容、注册门槛、价值、注意事项、来源和核验日期。灰区社区可以写内容类型，但不要写操作教程、交易路径或绕过方法。
