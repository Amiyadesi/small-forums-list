# 分类说明

这个仓库收录的是“小型 / 圈内 / 不主动搜索基本接触不到”的论坛和社区。这里的“收录”不是推荐或背书，只是做互联网社区地图。

目前内容主要由 AI 根据公开页面、搜索结果和圈内说法整理，只有少数社区来自长期个人体验。欢迎各社区成员提交 PR，补充真实氛围、注册状态、入口变化和更准确的分类。

## 灰区写法

灰区条目可以写清楚里面大概有什么，例如成人 ACG、游戏 MOD、网盘资源线索、求物、广告推广、账号/积分交易、盗版资源、擦边引流、钓鱼下载、隐私侵害、黑灰产信息等现象。

但仓库不写操作型内容：不追最新地址，不写绕过访问，不写资源下载步骤，不写交易路径，不写破解流程，不写诈骗或黑灰产操作教程。

写法上优先使用自然语言，不需要为了每个条目堆很多标签。JSON 里的 `risks` 字段只是给页面展示和后续筛选用，README 里更关注“这个社区主要干什么、进去前要知道什么”。

## 收录边界

- 不把 X、Instagram、贴吧、小黑盒、小红书、Reddit、Discord 或 Telegram 群本体列为主条目。
- 如果某个论坛有官方群、频道或社媒账号，只放在备注里，不作为主入口。
- 未亲自长期使用的社区，描述应写成公开信息和圈内观察，不伪装成亲身体验。

## 维护建议

- 每个条目至少保留一个官网、公开页面或稳定入口作为 `sources`。
- `last_checked` 使用 `YYYY-MM-DD`，表示最后一次人工核验入口和分类的日期。
- 新条目先进入 JSON，再运行 `node scripts/generate-readme.mjs` 同步 README。
- 修改页面展示后运行 `node scripts/build-site.mjs`，确认每个社区详情页能生成。
